ฐานข้อมูล SQLite

การกำเนิดของภาษา SQL (Structured Query Language)

ภาษา SQL (Structured Query Language) ถูกพัฒนาขึ้นครั้งแรกในช่วงต้นทศวรรษ 1970 โดยนักวิจัยของ IBM คือ Donald D. Chamberlin และ Raymond F. Boyce ที่ห้องวิจัยของ IBM ในซานโฮเซ รัฐแคลิฟอร์เนีย สหรัฐอเมริกา SQL ถูกสร้างขึ้นเพื่อใช้ในการเข้าถึงและจัดการข้อมูลในฐานข้อมูลแบบเชิงสัมพันธ์ (Relational Database) ซึ่งในตอนนั้นเรียกกันว่า SEQUEL (Structured English Query Language)

แรงบันดาลใจของ SQL มาจากทฤษฎีฐานข้อมูลเชิงสัมพันธ์ของ Dr. Edgar F. Codd ซึ่งตีพิมพ์ในปี 1970 บทความชื่อ “A Relational Model of Data for Large Shared Data Banks” ทำให้เกิดการปฏิวัติรูปแบบการจัดการข้อมูลจากเดิมที่ใช้ฐานข้อมูลแบบ hierarchical และ network มาเป็นแบบ relational แทน

ต่อมาในปี 1986 SQL ได้รับการยอมรับเป็นมาตรฐานโดย ANSI (American National Standards Institute) และในปี 1987 ได้กลายเป็นมาตรฐาน ISO (International Organization for Standardization) จนกลายมาเป็นภาษามาตรฐานที่ใช้ในการจัดการข้อมูลในฐานข้อมูลเชิงสัมพันธ์อย่างแพร่หลายในปัจจุบัน

ประวัติของ SQLite

SQLite (Structured Query Language Lite) เป็นฐานข้อมูลแบบฝังตัว (embedded database) ที่ไม่ต้องการการตั้งค่าเซิร์ฟเวอร์และใช้ทรัพยากรน้อย ถูกพัฒนาขึ้นโดย D. Richard Hipp ในปี 2000 จุดประสงค์หลักของ SQLite คือการเป็นฐานข้อมูลขนาดเล็กสำหรับการพัฒนาแอปพลิเคชันที่สามารถพกพาได้ โดยมีจุดเด่นหลักคือ:

  • ไม่มีเซิร์ฟเวอร์ (Serverless): SQLite ไม่จำเป็นต้องมีเซิร์ฟเวอร์หรือกระบวนการทำงานแบบแยกต่างหาก ข้อมูลจะถูกจัดเก็บในไฟล์เดียว (.sqlite) บนระบบไฟล์ของเครื่องผู้ใช้ ทำให้ง่ายต่อการพกพาและใช้งานในโปรเจกต์ขนาดเล็กและขนาดกลาง
  • ไม่ต้องติดตั้ง (Zero Configuration): SQLite สามารถเริ่มใช้งานได้ทันทีโดยไม่ต้องทำการติดตั้งหรือกำหนดค่าเพิ่มเติม
  • ขนาดเล็ก (Small Size): SQLite มีขนาดเล็กมากเมื่อเทียบกับระบบจัดการฐานข้อมูลอื่นๆ โดยไบนารีของ SQLite โดยทั่วไปมีขนาดประมาณไม่กี่ร้อย KB
  • เป็น Public Domain: SQLite ถูกปล่อยให้อยู่ใน public domain ซึ่งหมายความว่าใครก็ตามสามารถใช้มันได้โดยไม่มีข้อจำกัดทางลิขสิทธิ์

SQLite คืออะไร

SQLite เป็นไลบรารี่ที่ถูกสร้างโดยใช้ภาษา C เพื่อเป็นตัวจัดการฐานข้อมูล (database engine) สำหรับฐานข้อมูลเชิงสัมพันธ์ (relational database) ที่มีเอ็นจิ้นฐานข้อมูลแบบทรานแซคชั่นในตัวเอง (transactional) มีขนาดเล็ก (small) ทำงานได้เร็ว (fast) ทำงานได้ด้วยตนเอง (self-contained) เชื่อถือได้สูง (high-reliability) ไม่ต้องมีเซอร์ฟเวอร์ (serverless) ไม่ต้องตั้งค่าใดๆก่อนใช้งาน (zero-configuration) และมีความสามารถในการจัดการฐานข้อมูลด้วย SQL อย่างครบถ้วน (full feature) โดยความหมายของคำว่า lite หมายถึง ติดตั้งง่าย ดูแลระบบไม่ยุ่งยาก ไม่ต้องการทรัพยากรมาก และพร้อมใช้งานเป็นไลบรารีข้ามแพลตฟอร์มที่สามารถรวมเข้ากับภาษาโปรแกรมและระบบปฏิบัติการต่างๆ ได้อย่างไร้รอยต่อ

ข้อจำกัด

SQLite3 มีข้อจำกัดด้านขนาดของฐานข้อมูลและขนาดไฟล์ที่สามารถจัดเก็บได้ดังนี้:

ขนาดไฟล์ฐานข้อมูลสูงสุด (Maximum Database Size):

  • SQLite สามารถจัดเก็บข้อมูลในไฟล์ฐานข้อมูลได้สูงสุดที่ 140 TB (เทระไบต์) ต่อไฟล์ แต่ในทางปฏิบัติ ขนาดของไฟล์จะขึ้นอยู่กับระบบไฟล์และฮาร์ดแวร์ที่ใช้งาน เช่น ระบบไฟล์บางประเภทอาจมีข้อจำกัดที่ต่ำกว่านี้

ขนาดแถว (Row Size):

  • ข้อมูลในแต่ละแถวจะถูกจัดเก็บในรูปแบบของ blob โดยขนาดสูงสุดของ blob ใน SQLite สามารถมีได้ถึง 2 GB ต่อแถว

ขนาดของตัวแปรข้อความ (Text) และ Blob:

  • ขนาดสูงสุดของข้อมูลประเภท TEXT หรือ BLOB ที่สามารถจัดเก็บในหนึ่งแถวได้คือ 1 GB ต่อหนึ่งตัวแปร

จำนวนแถว (Number of Rows):

  • SQLite ไม่ได้มีข้อจำกัดตายตัวในเรื่องจำนวนแถว แต่จะขึ้นอยู่กับขนาดของไฟล์ฐานข้อมูลและหน่วยความจำที่ใช้งาน

จำนวนคอลัมน์ (Number of Columns):

  • ตารางใน SQLite สามารถมีคอลัมน์ได้สูงสุดที่ 32767 คอลัมน์ ต่อหนึ่งตาราง

จำนวนตาราง (Number of Tables):

  • ข้อจำกัดนี้จะขึ้นอยู่กับข้อกำหนดภายในของ SQLite และความจุของฮาร์ดแวร์ โดยในทางปฏิบัติสามารถมีจำนวนตารางได้หลายล้านตาราง

สำหรับการใช้งาน SQLite ในแอปพลิเคชันที่ต้องจัดการกับข้อมูลขนาดใหญ่ แนะนำให้พิจารณาความสามารถของฮาร์ดแวร์และระบบไฟล์ที่ใช้งานเพื่อให้สามารถจัดการฐานข้อมูลได้อย่างเหมาะสม

คุณสมบัติหลัก

ฐานข้อมูล SQLite เป็นแบบไฟล์และไม่ต้องการขั้นตอนการตั้งค่าที่ซับซ้อน ไฟล์เดียวทำหน้าที่เป็นฐานข้อมูลทั้งหมด ทำให้แจกจ่าย สำรองข้อมูล และถ่ายโอนได้ง่าย การไม่มีงานการดูแลระบบที่ซับซ้อนช่วยลดความยุ่งยากในการปรับใช้และการจัดการ ลดค่าใช้จ่ายในการบำรุงรักษาโดยรวม และเนื่องจากไม่ต้องมีเซอร์ฟเวอร์ จึงไม่ต้องมีการติดตั้งหรือตั้งค่าใดๆ ไม่ต้องมีกระบวนการเปิด/ปิด เซอร์ฟเวอร์ฐานข้อมูลก่อนการใช้งาน ไม่ต้องมีแม้แต่ไฟล์เพื่อกำหนดการตั้งค่า (configuration file)

บทสรุป

SQLite กลายเป็นโซลูชันฐานข้อมูลฝังตัวที่มีประสิทธิภาพและใช้งานได้หลากหลาย โดยนำเสนอแพลตฟอร์มที่มีน้ำหนักเบา สมบูรณ์ในตัวเอง และมีประสิทธิภาพสำหรับการจัดการข้อมูลที่มีโครงสร้างแก่นักพัฒนา ความเรียบง่าย พกพาสะดวก การรองรับแพลตฟอร์มที่หลากหลายทำให้เป็นตัวเลือกที่น่าสนใจสำหรับแอพพลิเคชั่นที่หลากหลาย ตั้งแต่ซอฟต์แวร์มือถือและเดสก์ท็อป

ในปัจจุบันมีระบบจัดการฐานข้อมูล SQL แบบฟรีหลายตัวที่ได้รับความนิยมและใช้งานอย่างแพร่หลายในทั้งงานโปรเจกต์ขนาดเล็กและขนาดใหญ่ นี้คือรายชื่อของฐานข้อมูล SQL แบบฟรีที่น่าสนใจ

MySQL / MariaDB MySQL เป็นระบบฐานข้อมูลแบบ open-source ที่มีประสิทธิภาพสูง รองรับการทำงานแบบ multi-user และ multi-threaded และได้รับการพัฒนาต่อเนื่องมายาวนาน โดย MariaDB เป็นฟอร์ค (fork) ที่แยกมาจาก MySQL หลังจากที่ MySQL ถูกซื้อโดย Oracle เพื่อคงความเป็นโอเพ่นซอร์สไว้ PostgreSQL PostgreSQL เป็นฐานข้อมูลที่เน้นการรองรับคุณสมบัติขั้นสูงตามมาตรฐาน SQL และการทำธุรกรรมแบบ ACID รองรับการจัดเก็บข้อมูลในหลายรูปแบบทั้งเชิงโครงสร้างและเชิงวัตถุ (Object-relational Database) Firebird Firebird เป็นฐานข้อมูลเชิงสัมพันธ์ที่มีขนาดเล็กและมีประสิทธิภาพสูง รองรับการทำธุรกรรมแบบ ACID และมีการพัฒนามาจากโค้ดเบสของ InterBase ซึ่งเป็นฐานข้อมูลที่เก่าแก่
